# 15 — App Review Submission

Status: Pending

## Goal

Submit a complete, reviewable App Store version.

## Final App Store Connect checklist

- [ ] Select the validated TestFlight build.
- [ ] Confirm app version and “What's New” text.
- [ ] Confirm screenshots and metadata match the build.
- [ ] Confirm privacy policy and support links are live.
- [ ] Confirm privacy labels are published.
- [ ] Confirm age rating, export compliance, content rights, pricing, and
      availability.
- [ ] Add App Review contact information.
- [ ] Add demo-account credentials.
- [ ] Explain camera scanning, AI processing/consent, shared pantries, and push
      reminders in Review Notes.
- [ ] Include any steps needed to trigger non-obvious functionality.
- [ ] Confirm Supabase, Edge Functions, cron, authentication providers, and push
      delivery remain live during review.
- [ ] Submit for review.

## Rejection handling

- [ ] Record the exact guideline and reviewer message.
- [ ] Reproduce the issue against the submitted build.
- [ ] Respond with concrete evidence or submit a corrected build.
- [ ] Update the relevant plan step so the issue does not recur.

## Done when

Apple approves the version and it is released according to the selected release
method.
